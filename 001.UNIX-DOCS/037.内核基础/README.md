## 内核分类
|内核分类|说明|代表|特点|
|-|-|-|-|
|宏内核(Monolithic Kernel)|- 理念： “大而全”。将操作系统的大部分功能（进程管理、内存管理、文件系统、设备驱动、网络协议栈）全部集成在内核中 <br/>|- Linux <br/>|- 优点: <br/> + 高性能 |
|-|-|-|-|
|微内核(Micro Kernel)|- 设计原则：最小化内核功能 <br/> - 工作方式： 将文件系统、驱动程序、网络等功能剥离出来，放在用户态（User Space）运行，称之为“服务进程”。内核只充当“邮递员”，负责转发各服务之间的消息|- HarmonyOS NEXT（原生鸿蒙）|- 优点: <br/> + 可靠性高 |
|-|-|-|-|
|- 外核(Exo Kernel) <br/> 另一种外核： Uni-Kernel(单内核)|- 不管理资源，只管理应用 <br/> 操作系统 = 服务应用<sup>用户态:libOS ， 即应用直接调用提供的库(Library)</sup> + 管理应用<sup>内核态: ExoKernel</sup>|-|- 将时间片切片<br/> - 在一个切片内，系统只服务一个应用<br/> -该应用可以访问所有硬件资源<br/>|
|-|-|-|-|
|- 多内核(Multi-Kernel)|- 多核与异构（一个芯片上有 GPU + CPU + NPU<sup>Neural Processing Unit: 神经网络处理单元</sup>）|-|-|
