- 操作系统作为一个软件在CPU上运行，他必须保证自己读取（str load）存储操作，他要保证自己用的是虚拟地址，那就需要一个物理地址转换为虚拟地址的过程。
  + 图中“L3页表项”存储的是物理地址，若操作系统想要`对整个页表做修改`<sup>操作系统修改或创建一个新的页表</sup>，必须保证图中“L3页表项”是物理地址
  + 操作系统作为一个软件在CPU上运行，他必须保证自己读取/存储<sup>也要执行str load 等汇编指令</sup>操作使用的是虚拟地址
  + 那就需要一个物理地址转换为虚拟地址的过程：直接将物理地址 + (物理地址和虚拟地址的偏移量)

## 地址翻译的"单向性"与"终点"
当 CPU 开启 MMU 后，执行一条指令（例如 LDR R0, [R1]，其中 R1 存的是虚拟地址）时，硬件内部发生的步骤:
|步骤|说明|备注|
|-|-|-|
|- 1. |- CPU 核心 (Core)：计算出需要访问的虚拟地址 (VA)。|-|
|- 2. |- 发送请求：CPU 核心将这个 VA 发送给旁边的 MMU 单元|-|
|- 3. |- MMU 查表/TLB|-|
|- 4. |- 输出物理地址|- MMU输出这个物理地址|
|- 5. |- 内存访问|- 这个物理地址被放到系统总线上，发送给内存控制器,CPU核心在地址被发出后，就等待数据返回了，它不会再把这个地址重新解释为虚拟地址|
|-|-|-|
