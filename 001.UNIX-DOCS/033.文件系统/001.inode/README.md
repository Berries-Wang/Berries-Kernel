# inode

## Why 4096 Bytes?
在操作文件的时候(读取/写入)，Linux 内核并不是直接对文件进行操作，而是以块(block)为单位进行处理。每个块的大小通常是 4096 字节(4KB)<sup>这个数值是磁盘空间利用率与系统性能之间权衡折衷的结果: 块越小，需要的块越多，所需维护块辅助数据结构越多</sup>。
- 效率：以块为单位进行读写操作，可以减少磁盘寻址的次数，提高数据传输效率。
- 内存对齐：4KB 是大多数现代处理器的内存页大小，有助于内存管理和缓存优化。
- 文件系统设计：许多文件系统(如 ext4)选择 4096 字节作为块大小，以平衡存储效率和性能。

### What 4096 Bytes? <sup>数据块(Data Block)大小： 具体根据配置，不一定是4096</sup>
在 Linux 文件系统中，数据块（Data Block）是磁盘空间分配和 I/O 读写的最小基本单位

        File: grub.cfg
        Size: 117       	Blocks: 8          IO Block: 4096(数据块大小)   regular file
